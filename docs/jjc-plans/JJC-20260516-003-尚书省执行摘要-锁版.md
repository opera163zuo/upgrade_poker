## 三、牌型强度全层级表（新增类型用 **粗体** 标出）

```
类别      类型层级      说明
─────────────────────────────────────
单张      Level 1     1张
对子      Level 2     2张同花色同点
三张      Level 3     3张同花色同点（独立牌型） ← 新增
对子拖拉机 Level 4     每点位2张（连续点数）
四张(单点) Level 5     4张同花色同点（Quad）  ← 新增
三张拖拉机 Level 6     每点位3张（连续点数）   ← 新增
四张拖拉机 Level 7     每点位4张（连续点数）   ← 新增
```

**核心比较规则**：先比级别（7>6>5>4>3>2>1），同级别比连续点数长度，同长度比最大点数。

**特判**：Level 5 (四张单点) > Level 4 (对子拖拉机)，但 < Level 6 (三张拖拉机)

## 四、底牌倍率计算（必须测试）

```
倍率公式: 2^(最后一墩赢家出牌中的对子总数 + 1)

出牌类型     → 对子数 → 倍率
单张         → 0     → ×2
对子         → 1     → ×4
三张         → 1     → ×4
四张         → 2     → ×8
对子拖拉机(2对) → 2  → ×8
对子拖拉机(3对) → 3  → ×16
三张拖拉机(2组) → 2  → ×8
四张拖拉机(2组) → 4  → ×32
```

## 五、甩牌新规则（代码注意事项）

```
输入: 同花色多组件牌组（如 A + KK + QQQ）
处理:
  1. splitIntoComponents() — 拆成独立组件
  2. 对每组件检查: canBeBeat(comp, allOpponents)
     - 同花色同类型比较
     - 仅检查最大可能性（最坏情况对手有压牌）
  3. 若全部安全 → 全部打出
  4. 若任一不安全 → 处罚模式:
     - 筛选安全组件列表
     - 选"最小安全组件"（类型最低→张数最少→点数最小）
     - 仅打出该组件
     - 其余全部收回手牌
     - 其他玩家按该组件类型跟牌
```

**⚠️ 关键实施细节**：
- `canBeBeat` 需检查**所有其他三家**手牌的联合能力
- 组件类型匹配：单张只能被同花色更大单张压
- 四张(单点)可以被更大四张压，或同等条件下更大的点数
- 处罚模式下打出的"最小安全组件"决定了本轮的游戏走势
