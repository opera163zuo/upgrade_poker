## 三、太子方案复核（完整性检查）

### 3.1 总体方案评价

**结论：方案方向正确，但存在重大遗漏和隐式假设。**

#### ✅ 方案优势

1. **分阶段思路正确**：先做最小可运行，再做完整改造
2. **已有架构分析深入**：Workspace-taizi 中的三层拆解文档详尽
3. **参考实现充分**：Rust 版 4 副牌可编译运行，为 Go 版改造提供了对照蓝本
4. **第一批改动关注点合理**：UI状态面板和基础规则修复是改造前置条件

#### ❌ 方案缺陷

1. **未明确定义"四副牌"的完整规则**——四副牌不是两副牌的简单加倍，有独特的拖拉机判定规则（三连对以上、四张同牌作为特殊牌型等）
2. **缺少发牌模型的形式化定义**：216张牌如何分配到4家（52×4 + 8），与108张（25×4 + 8）的差异如何影响布局
3. **未讨论52张手牌在640×480画布上的显示方案**：这是最大的UI挑战
4. **缺少 AI 升级方案**：52张手牌导致搜索空间爆炸
5. **缺少"四张同牌"作为拖拉机单元的处理**：这在4副牌中是关键区别
6. **底牌得分倍率未定义**：4副牌下底牌分 ×4 还是 ×2？
7. **局数/升级规则未定**：从2开始打到A算一轮还是需循环？

## 五、缺失规则定义清单

经比对 C# 原版（基线A）与 Rust 版（基线B），以下规则在 Go 版中**未定义或定义不完整**：

### 5.1 四副牌特有牌型

| 牌型 | 二副牌 | 四副牌 | 当前 Go 支持 | 改造要求 |
|------|--------|--------|-------------|---------|
| 单张 | ✓ | ✓ | ✓ | 无需改动 |
| 对子 | 同花同点×2 | 同花同点×2/3/4中任取2 | ✓（但假设最多2张） | 需支持从4张中选择2张作为对子 |
| 三连对(拖拉机) | 同花色3个连续对 | 同花色3+个连续对 | ✓（假设每对只2张） | 需适配每级可多达4张的情况 |
| 四连对(拖拉机) | 不存在（最大3连） | 存在 | ✗ | 新增支持 |
| **四张同牌(炸弹)** | 不存在（不可能） | **新牌型**，同花色同点4张 | ✗ | **新增支持** |
| **四张拖拉机** | 不存在 | 同花色连续4张同牌可作为拖拉机变体 | ✗ | 需定义：四张K-K-K-K是否为拖拉机 |
| 混合拖拉机 | 对子+单张混出 | 对子+单张+四张混出 | ✗（对子+单张混出不支持） | 需扩展甩牌规则 |

### 5.2 四级牌（四张同牌）的处理规则

这是4副牌最关键的缺失定义点。参考原版逻辑：

```
a. 四张同花色同点数牌（如♠K×4）构成特殊单元
b. 可作为"拖拉机"对待，即等同于一个"双对子"
c. 在毙牌时，需要同是四张同牌才能盖毙
d. 四张同牌 > 拖拉机 > 对子 > 单张（在毙牌比较时）
```

### 5.3 跟牌强制规则（有对必出对）

4副牌下的复杂性：

```
场景：首家出♠A（单张）
我手中有：♠A×2、♠K×1
问题：我是否有♠A这个对子？
→ 传统规则：有对必出对，但我手中同一花色的♠A只有2张，视为对子
→ 四副牌变体：2张即构成对子，即使同一牌在别人手中还有另外2张

场景：首家出♠KK（对子）
我手中有：♠K×4
问题：我是只能出一对♠K，还是必须出两对？
→ 规则差异：多数四副牌规则要求出最大数量的对子
→ 建议：以手牌实际构成的最大对子数上跟
```

### 5.4 毙牌规则定义（4副牌特有）

```
毙牌要求：
1. 必须出主牌
2. 主牌的"结构"必须不少于首家的"结构"
   - 首家出单张 → 毙牌出≥1张主牌
   - 首家出对子 → 毙牌出≥1对主牌
   - 首家出拖拉机 → 毙牌出≥1组相同长度的拖拉机
   - 首家出四张同牌 → 毙牌需出相同张数的主牌（或更大的四张同牌）

当多家同时毙牌时（盖毙）：
- 比较各毙牌的主牌大小
- 拖拉机 > 对子 > 单张（同级数比较下的优先级）
```

### 5.5 计分规则差异

| 项目 | 二副牌（当前Go） | 四副牌 | 建议 |
|------|---------------|-------|------|
| 总分 | 200分 | **400分** | 5/10/K 各4倍 |
| 底牌倍率 | ×2 | 待定 | 建议保留×4倍底牌再×2（与原C#版一致） |
| 升3级阈值 | <40分 | <80分或保持<40？ | **高风险争议点**。两种常见规则：①全部双倍（<80升3级）；②缩小级差（<40升3级但等级更多） |
| 降级阈值 | ≥120分 | ≥240分或保持≥120 | 需与升级阈值统一 |
| 游戏结束 | 打到A后再打2 | 打到A后再打2（与二副相同） | 这个不改 |

### 5.6 叫主/反主规则

| 情形 | 二副牌 | 四副牌 | 建议 |
|------|--------|--------|------|
| 亮单张级牌 | 允许 | 通常不允许或弱优先 | 建议保留单亮，但可被对亮/三亮覆盖 |
| 亮对级牌 | 确定主花色 | 确定主花色 | 不变 |
| 亮三张级牌 | 通常不允许（只有2副） | 覆盖对级牌 | **新增规则** |
| 亮四张级牌 | 不存在 | 最高权重 | **新增规则** |
| 亮对大王/小王 | 无主 | 无主 | 不变 |
| 无人亮主 | 打无主 | 打无主 | 不变 |
